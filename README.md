# gogcli-mcp

MCP server that exposes [gog](https://gogcli.sh) CLI commands as tools for [Claude Co-Work](https://claude.ai) and other MCP clients.
Used to connect Claude Co-Work to one of more Google Workspace accounts in read/write or readonly mode per service.
MacOS only.

## Why

Co-Work's built-in Google connectors only support a single Google account and are limited to Calendar and Gmail. This server removes both restrictions — connect multiple Google accounts and access the full range of Google Workspace services: Gmail, Calendar, Drive, Docs, Sheets, Slides, Contacts, Tasks, Chat, Forms, Keep, Classroom, Groups, Admin, and Apps Script. Authorization can be scoped to read-only access per service via `gog auth add --readonly`.

## Quick Start

### 1. Install gog and authorize a Google account

```bash
brew tap steipete/tap
brew install gogcli
gog auth add --service gmail,calendar,drive,tasks
```

See [gog docs](https://gogcli.sh) for the full list of services and `--readonly` scoping.

### 2. Set up a Cloudflare Tunnel

Co-Work cannot connect to localhost. You need a tunnel to give the server a public URL. Cloudflare Tunnel is recommended — it's free to use, you just need a domain (~$10/yr) with DNS on Cloudflare.

```bash
brew install cloudflared

# Authenticate with Cloudflare (opens browser)
cloudflared tunnel login

# Create a tunnel
cloudflared tunnel create gogcli-mcp

# Route your domain to the tunnel
cloudflared tunnel route dns gogcli-mcp gog.yourdomain.com
```

Test it manually before proceeding:

```bash
# Terminal 1: start the server
GOG_MCP_TRANSPORT=http GOG_MCP_ISSUER=https://gog.yourdomain.com ./gogcli-mcp

# Terminal 2: start the tunnel
cloudflared tunnel run --url https://localhost:9247 --no-tls-verify gogcli-mcp
```

Verify: `curl https://gog.yourdomain.com/.well-known/oauth-authorization-server` should return JSON.

> **Note:** `--no-tls-verify` is required because the server uses a self-signed cert on localhost. Cloudflare terminates real TLS at the edge.

> **Other tunnel options:** [Tailscale Funnel](https://tailscale.com/kb/1223/funnel) also works (free, URL is `machine.tailnet.ts.net`). ngrok free tier is **not compatible** — its browser interstitial breaks the OAuth flow.

### 3. Install gogcli-mcp and run setup

```bash
brew tap flythebluesky/tap
brew install gogcli-mcp
gogcli-mcp setup
```

The wizard verifies gog is working, generates OAuth credentials, asks for your public URL, and installs a launchd service that starts automatically on login.

### 4. Connect Co-Work

The wizard prints credentials like:

```
Server:    https://gog.yourdomain.com
Client ID: a1b2c3d4e5f6...
Secret:    x9y8z7w6v5u4...
```

1. In Co-Work, go to **Settings > Connectors > Add custom connector**
2. Enter URL: `https://gog.yourdomain.com/mcp` (the `/mcp` suffix is required)
3. Click **Advanced settings**
4. Paste the Client ID and Secret
5. Click **Add**
6. Approve access in the browser when prompted

## Authentication

The server implements OAuth 2.1 per the [MCP authorization spec](https://modelcontextprotocol.io/specification/2025-03-26/basic/authorization). No open registration — clients must use a pre-registered client ID and secret (generated by the setup wizard).

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/.well-known/oauth-authorization-server` | GET | Metadata discovery (RFC 8414) |
| `/authorize` | GET/POST | Browser consent page + authorization code |
| `/token` | POST | Token exchange with PKCE S256 + refresh tokens |
| `/mcp` | POST | MCP endpoint (requires Bearer token) |

Token endpoint supports both `client_secret_post` and `client_secret_basic` authentication.

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `GOG_PATH` | `gog` | Path to the gog CLI binary |
| `GOG_MCP_TRANSPORT` | (stdio) | Set to `http` to enable HTTPS server mode |
| `GOG_MCP_ADDR` | `:9247` | Listen address |
| `GOG_MCP_ISSUER` | `https://localhost:<port>` | Public URL for OAuth metadata |
| `GOG_MCP_CLIENT_ID` | | Pre-register an OAuth client with this ID on startup |
| `GOG_MCP_CLIENT_SECRET` | | Pre-register an OAuth client with this secret on startup |
| `GOG_MCP_TLS_CERT` | (auto-generated) | Path to custom TLS certificate |
| `GOG_MCP_TLS_KEY` | (auto-generated) | Path to custom TLS private key |
| `GOG_MCP_CONFIG_DIR` | `~/.config/gogcli-mcp` | Config directory (certs, OAuth state, tunnel config) |

## Reusable OAuth Package

The OAuth layer lives in `pkg/mcpauth/` and can be used by any Go MCP server:

```go
import "gogcli-mcp/pkg/mcpauth"

store := mcpauth.NewStore("/path/to/oauth.json")
client := store.RegisterClient("My App", []string{"https://example.com/callback"})

handler := mcpauth.NewHandler(store, "https://my-issuer.com")

mux := http.NewServeMux()
mux.Handle("/.well-known/oauth-authorization-server", handler)
mux.Handle("/authorize", handler)
mux.Handle("/token", handler)
mux.Handle("/mcp", mcpauth.Middleware(store, "https://my-issuer.com", yourMCPHandler))
```

## Limitations

- **macOS only.** The setup wizard installs a launchd service. Linux systemd support is deferred.
- **Single user.** No user database. The consent page approves access for whoever clicks "Approve." Designed for personal use.
- **Token lifecycle.** Access tokens expire after 1 hour and are automatically renewed via refresh tokens (valid 1 year). Refresh tokens are rotated on each use.
- **No token revocation endpoint.** To revoke all tokens, delete `~/.config/gogcli-mcp/oauth.json` and restart the server.
- **Self-signed certs.** Auto-generated certs are self-signed. Cloudflare terminates real TLS at the edge when using a tunnel.

